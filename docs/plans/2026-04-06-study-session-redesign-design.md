# 学习会话页重构设计

## 目标

重构点击“新学 / 复习”后进入的学习会话页，提升视觉层级、操作反馈和完成态表达，同时保持现有学习队列与判定逻辑不变。

## 范围

- 仅修改 `StudySessionPage` 及其直接相关测试
- 保留现有 `StudySessionController` 的决策与推进逻辑
- 保留“标记熟悉”顶部入口，以及底部“忘记 / 模糊 / 认识”三个按钮
- 完善来源标识、进度、状态反馈、完成态摘要

## 方案

### 1. 顶部信息区

- 使用统一的会话头部卡片，展示返回、当前进度、来源标签（新学 / 复习 / 抽查）
- 顶部右侧保留“标记熟悉”，但改成更稳定的操作按钮
- 增加整体进度条，让用户能感知本轮完成进度

### 2. 单词主卡区

- 单词、音标、词性/释义入口集中在主卡片内
- 单词字号更大，信息分组更清晰
- 定义未展开时保持聚焦单词；展开后展示释义与查看完整释义入口

### 3. 底部操作区

- 三个判断按钮统一为大触达按钮，颜色语义清晰
- 增加当前按钮按压态与轻量反馈文案
- 顶部“标记熟悉”和底部三按钮在统计口径上继续保持既有语义

### 4. 完成态

- 完成页改成结果卡片式摘要
- 展示本轮统计：已掌握、认识、模糊、忘记
- 增加总结文案与返回按钮

## 测试策略

- 保留现有会话页行为测试
- 新增/调整 widget tests：
  - 顶部进度与来源标签可见
  - 底部三个按钮可见且能推进
  - “标记熟悉”进入完成态统计正确
  - 完成页展示新的摘要结构

## 风险与约束

- 当前会话中存在其他未提交改动，本次仅在学习会话页相关文件中推进，避免误伤
- 不修改 `StudySessionController` 的核心学习决策，降低回归风险
