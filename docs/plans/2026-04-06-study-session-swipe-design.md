# 学习会话页左右滑卡交互设计

## 目标

为当前学习会话页增加 Tinder 式左右滑卡交互，在卡片模式下支持：

- 左滑判定“忘记”
- 右滑判定“认识”

同时保留现有底部按钮与顶部“标记熟悉”入口，不引入“模糊”手势。

## 范围

- 修改 `lib/features/study/presentation/study_session_page.dart`
- 调整学习会话相关 widget test
- 不修改 `StudySessionController` 评分语义
- 不改设置页文案与 `StudyMode` 枚举定义
- 不为“模糊”增加手势

## 交互方案

### 1. 手势区域

- 仅主单词卡支持横向拖拽
- 详情入口、例句区、底部按钮继续保留现有点击行为

### 2. 拖拽反馈

- 卡片跟随手指做水平位移
- 同步添加轻微旋转，形成 Tinder 式动态反馈
- 左右方向分别展示“忘记”“认识”的状态提示

### 3. 触发时机

- 以松手后的最终位移决定是否触发
- 超过阈值时，卡片飞出屏幕，再执行对应判定
- 未超过阈值时回弹，不触发任何学习记录

### 4. 状态约束

- `_isSubmitting` 为 `true` 时禁用滑动与按钮重复触发
- 触发手势判定后仍统一复用 `_handleDecision()`，保持现有震动反馈与防重逻辑

## 测试策略

- 新增右滑触发“认识”的 widget test
- 新增左滑触发“忘记”的 widget test
- 新增未过阈值时仅回弹、不记录判定的 widget test

## 风险与约束

- 当前学习页尚未拆出独立的非卡片模式页面，本次仅在现有卡片会话页接入手势
- 需控制手势作用范围，避免和页面滚动、详情点击产生明显冲突
